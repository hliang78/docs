# OneOPS RCA 下一阶段开发边界清单

本文档只定义下一阶段边界。

不补充机制新能力。
不展开真实数据执行。
不做泛化规划。

---

## 1. 当前阶段状态

当前已经固定的基线只有四项：

1. `pkg/rca` 平台无关机制已抽离
2. OneOps 适配层边界已收口
3. 平台无关夹具与 OneOps 伪集成夹具已落地
4. 统一回归脚本已可执行并通过

因此，下一阶段不再以“继续拆代码”为目标。

---

## 2. 下一阶段只允许做的事

下一阶段只允许做以下三类工作：

1. 维护当前夹具
2. 维护当前入口回归
3. 维护当前文档边界

若出现以下情况，允许做最小修正：

- 夹具失效
- 回归脚本失效
- 适配层字段映射与文档不一致
- 已有主线行为被破坏

修正要求：

- 只修当前主线
- 不顺手扩能力
- 不引入新判定维度

---

## 3. 下一阶段明确不做的事

下一阶段明确不进入以下内容：

1. 排序
2. 置信度
3. 历史因果分析
4. 变更关联建模
5. 新增独立顶级模块
6. 新的 RCA 输出形态
7. 围绕“智能化”展开的新设计
8. 真实数据执行本身
9. 把动画展示层做成机制能力的一部分
10. 路径组模型

若未来单独启动动画展示，也只能满足以下边界：

- 只作为结果播放层
- 只消费当前 RCA 稳定输入输出
- 不改 `pkg/rca` 判定逻辑
- 不阻塞当前基线维护

若未来单独启动“路径组语义”，也只能满足以下边界：

- 只解决“部分分支阻断、其他分支仍存活”的表达缺口
- 不改当前场景 JSON 主线字段含义
- 不扩成通用路径建模平台
- 不先于真实场景需求落地

---

## 4. 若必须继续开发，只能沿这条顺序推进

若未来仍需继续代码开发，顺序固定为：

1. 先修基线失败
2. 再修夹具与文档不一致
3. 最后才考虑真实数据接入前的最小准备

不允许跳过前两步直接进入新开发。

---

## 5. 下一阶段完成标准

下一阶段只在以下条件同时满足时视为完成：

1. 基线脚本持续可执行
2. 平台无关夹具持续通过
3. OneOps 伪集成夹具持续通过
4. API 入口回归持续通过
5. 三份主文档边界保持一致

---

## 6. 下一阶段执行清单

下一阶段只按下面顺序执行：

1. 先运行 `OneOps/scripts/run_rca_fixture_checks.sh`
2. 若脚本失败，只修失败点
3. 若脚本通过，只检查文档与夹具是否仍一致
4. 若文档与夹具一致，本阶段不再继续开发

本阶段每日检查也只允许记录三项：

1. 基线脚本是否通过
2. 夹具是否有新增失效
3. 文档边界是否被破坏

不追加新的检查维度。

---

## 7. 下一阶段结束后，才允许进入的事

只有在业务上明确开启真实数据窗口后，才允许进入：

1. 真实租户验收
2. 真实告警验收
3. 真实日志证据验收
4. 真实阻塞点记录

届时记录模板以：

- [ONEOPS_RCA_REAL_ACCEPTANCE_TEMPLATE.md](/home/jacky/project/OneOPS-ALL/docs/ONEOPS_RCA_REAL_ACCEPTANCE_TEMPLATE.md)

为准。

阶段交接以：

- [ONEOPS_RCA_PHASE_HANDOFF.md](/home/jacky/project/OneOPS-ALL/docs/ONEOPS_RCA_PHASE_HANDOFF.md)

为准。
