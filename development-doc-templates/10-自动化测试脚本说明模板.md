## 5. 响应验证规则

HTTP 状态码为 200 不代表业务处理成功，必须同时验证响应体中的业务状态字段。

### 5.1 兼容响应字段

| 字段 | 成功条件 | 说明 |
|------|----------|------|
| `success` | `true` | 优先使用 |
| `code` | `200` 或 `0` | 兼容通用响应 |
| `responseCode` | `0` | 兼容业务响应 |

### 5.2 标准响应结构

```json
{
  "timestamp": 1700000000000,
  "responseCode": 0,
  "success": true,
  "message": "操作成功",
  "errorMsg": null,
  "data": {}
}
```

## 6. 测试脚本结构

| 区域 | 内容 | 说明 |
|------|------|------|
| 导入区 | `pytest`、`requests`、`json`、`time` | 脚本依赖 |
| 配置变量区 | `BASE_URL`、`JWT_TOKEN`、`HEADERS` | 运行环境配置 |
| 响应验证器 | `ResponseValidator` | 统一成功/失败断言 |
| 测试数据区 | `TestData` 或外部 JSON | 正常、边界、异常数据 |
| 测试用例类 | `Test{模块名称}` | 按模块组织 |
| 报告输出 | pytest-html | 输出 HTML 报告 |

## 7. 测试用例映射

| 测试用例编号 | pytest 方法名 | 接口 | 测试类型 | 断言重点 |
|--------------|---------------|------|----------|----------|
| TC_W_001 | `test_tc_w_001_create_success` | `{接口}` | 写入成功 | HTTP 状态、业务成功、返回数据 |

## 8. pytest 标记

| 标记 | 说明 |
|------|------|
| `@pytest.mark.smoke` | 冒烟测试 |
| `@pytest.mark.write` | 写入类接口 |
| `@pytest.mark.query` | 查询类接口 |
| `@pytest.mark.boundary` | 边界测试 |
| `@pytest.mark.permission` | 权限测试 |

## 9. 执行命令

```bash
pytest tests/test_{模块名称}.py -v --html=reports/test_report_{模块名称}_{执行时间}.html --self-contained-html
```

## 10. 报告要求

| 报告项 | 要求 |
|--------|------|
| 执行时间 | 记录开始、结束和耗时 |
| 通过率 | 统计通过、失败、跳过数量 |
| 失败详情 | 记录请求参数、响应内容、断言错误 |
| 环境信息 | 记录 BASE_URL、模块名称、版本号 |

## 11. 注意事项

- 测试数据应避免污染生产或共享环境。
- 写入类测试建议包含清理逻辑或使用隔离测试数据。
- token 不应提交真实生产凭证。
- 业务失败场景需要验证错误消息是否符合预期。
