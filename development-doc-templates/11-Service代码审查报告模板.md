## 3. 审查标准

| 分类 | 审查项 |
|------|--------|
| 命名规范 | public 方法是否符合查询、统计、聚合、复杂查询命名规则 |
| 基础代码质量 | 是否存在无用变量、不可达分支、无用 try-catch |
| 参数校验 | public 方法入口是否完成非空、长度、范围、集合校验 |
| 异常处理 | 是否合理处理异常，避免只打印堆栈或吞没异常 |
| 事务管理 | `@Transactional` 使用位置、回滚行为、只读事务、传播行为 |
| 并发与性能 | 线程池使用、缓存设计、大对象处理、查询效率 |
| 注释规范 | public 方法是否有完整 JavaDoc |

## 4. 方法命名规范检查

| 方法类型 | 命名规范 | 示例 |
|----------|----------|------|
| 单条件查询 | `findBy{FieldName}` | `findByUserId(Long userId)` |
| 多条件查询 | `findBy{Field1}And{Field2}` | `findByProvinceCodeAndCityCode` |
| 统计查询 | `countBy{条件}` | `countByStatus(Integer status)` |
| 求和/聚合 | `sumBy{字段}` | `sumByAmount(Long orderId)` |
| 多条件复杂查询 | `{方法名}ByConditions` | `findByConditions(OrderQueryDTO dto)` |
| 通用查询 | `findAll`、`findById` | `findById(Long id)` |
