# FE 运维入口适配

## Purpose

在 Device V2 清册页增加单设备运维入口，并保证所有旧能力调用前都经过 V1 code gate。

## Findings

- 行级 `更多` 当前没有监控/WebShell/操作审计/日志。
- 详情侧栏 `操作中心` 当前只有维护设备资料。
- 旧 WebShell 使用 `TerminalFull` query：`hostCode`、`protocol`、`hostname`。
- 旧操作审计使用 `sshRecordPageReq`，参数 `instance_code`。

## Requirements

- 封装 `ensureLegacyDeviceForOps` 或同等集中函数。
- 入口不得从 V2 attributes 私自猜测 V1 code；必须使用后端 bridge/status/sync 结果。
- 所有失败显示后端原因或缺字段列表。
- 日志入口显示待确认，不发起 Loki 查询。

## Acceptance

- Row `更多` 有新增入口。
- 详情侧栏操作中心有新增入口。
- WebShell 和审计使用 V1 code。
- 旧清册页行为不变。

## Validation

- `cd OneOPS-UI && npm run typecheck:d2`
- Browser evidence with selected V2 device.

- D2V2-002, D2V2-003, D2V2-004, D2V2-005, D2V2-006
