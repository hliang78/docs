# BE V1 同步与旧能力契约

## Purpose

让 Device V2 前端能明确知道单台设备是否已桥接到 Device V1，并在需要时调用同步能力。

## Findings

- 现有 `sync-to-v1` 是批量接口。
- 后端会回填 `device_v1_code`、`legacy_device_code`、`sync_to_v1_status`。
- 前端没有显式单设备 bridge 状态契约。

## Requirements

- 增加或确认单设备 bridge/status/ensure 契约。
- 复用现有同步逻辑，避免并行新建第二套同步。
- 缺少 `platform_code`、`catalog_code`、`site_code`、`rack_code` 等 V1 必填事实时直接报错。
- 不改变现有批量同步行为。

## Acceptance

- Go 测试覆盖已同步、创建/回填、缺字段失败。
- API response 对前端足够明确。
- 不添加 mock 或静默成功。
