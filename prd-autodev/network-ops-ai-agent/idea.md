# Idea

## Original Input

网络运维智能体：基于AI大模型的自然语言一问一答交互，内置网络、安全类的设备参数&运维知识库，通过现象描述（如日志告警、设备异常）进行根因分析、提供故障排查步骤及解决方案；自动学习机制：对上传的用户私有知识文档进行智能解析、语义理解与知识抽取，并将其无缝融入现有AI问答体系。希望在 OneOPS 中融入这些内容，关键组件可考虑利用第三方开源平台。

## Known Constraints

- OneOPS 现有 RCA、设备、拓扑、采集、日志能力不能被外部平台替换，只能被编排和增强。
- 第一阶段以只读诊断和建议为主，不做自动修复闭环。
- AI 输出必须保留证据来源，并区分事实、推断、建议和缺口。
- 私有知识文档必须保留租户、权限、来源、版本、质量状态。
- 第三方开源平台优先作为 PoC 或旁路服务，长期内核需可替换。
