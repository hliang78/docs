# Platform 脚本执行前端重构 - Structure Proposal

## 目标

把当前 4 个页面从“功能堆叠”改成“工作流分工明确”的产品结构，让用户更快回答这 3 个问题：

1. 我现在应该去哪个页面
2. 我在这个页面应该先做什么
3. 做完这一步之后下一步去哪里

## 一句话定位

- `脚本模板工作台`：创作模板
- `任务模板管理`：管理模板目录与默认配置
- `任务中心`：发起和追踪执行
- `定时任务管理`：配置周期调度

## 新职责图

### 1. 脚本模板工作台

**定位**

模板创作工作台，只负责把想法变成可复用模板。

**应该保留的能力**

- 创建 / 打开会话
- 多轮对话生成草稿
- 草稿编辑
- 草稿版本管理
- 本地校验
- 发布为模板

**应该降级的能力**

- 真实任务执行测试
  当前可保留，但应作为“高级验证方式”，不应与主创作流程并列

**不应该承担的角色**

- 日常任务执行入口
- 周期调度入口
- 任务结果总览入口

### 2. 任务模板管理

**定位**

模板目录页，负责“找模板”和“维护默认配置”。

**应该保留的能力**

- 模板列表
- 搜索 / 类型筛选
- 模板描述与适用场景
- 默认执行配置编辑
- 从模板去创建任务
- 从模板去创建定时任务

**应该强化的能力**

- 模板可读性
  不只是名字，还应快速看出它适用于什么、默认怎么跑、依赖什么

**不应该承担的角色**

- 复杂脚本创作
- 执行日志查看

### 3. 任务中心

**定位**

执行中心，负责“跑一次”和“看结果”。

**应该保留的能力**

- 常用执行入口
- 最近任务列表
- 状态与日志查看
- 新建任务
- 复用历史任务
- 停止运行中任务

**应该强化的能力**

- 分步创建任务
- 执行前摘要确认
- 最近运行状态可扫读

**不应该承担的角色**

- 模板长期维护
- 周期调度配置

### 4. 定时任务管理

**定位**

调度中心，负责“让模板定期跑起来”。

**应该保留的能力**

- 新建 / 编辑计划
- Cron 配置
- 启用 / 禁用
- 最近运行跳转
- 少量覆盖默认执行配置

**应该强化的能力**

- 调度状态可读性
- 最近运行成功率 / 最近失败
- 覆盖字段与模板继承边界说明

**不应该承担的角色**

- 复杂模板配置
- 全量执行参数主编辑

## 页面关系图

```text
[脚本模板工作台]
  产出模板
      |
      v
[任务模板管理]
  选模板 / 维护默认配置
   |                    \
   |                     \
   v                      v
[任务中心]            [定时任务管理]
  跑一次 / 看结果        配周期 / 看最近运行
```

## 推荐主导航关系

```text
一级主线：
1. 模板创作
2. 模板目录
3. 执行中心
4. 调度中心

辅助跳转：
- 模板目录 -> 执行中心：基于模板跑一次
- 模板目录 -> 调度中心：基于模板建计划
- 执行中心 -> 模板目录：查看模板来源
- 调度中心 -> 执行中心：查看某次运行结果
```

## 关键用户路径

### 路径 A：我要从零做一个模板

```text
进入脚本模板工作台
-> 创建会话
-> 对话生成草稿
-> 编辑与校验
-> 发布成模板
-> 跳到任务模板管理查看模板
```

### 路径 B：我要用现有模板立刻跑一次

```text
进入任务中心
-> 选择推荐模板或新建任务
-> 选择目标范围
-> 填少量参数
-> 执行前确认
-> 查看任务状态与日志
```

### 路径 C：我要让某个模板每天定时跑

```text
进入定时任务管理
-> 选择模板
-> 配项目 / 区域 / Cron
-> 必要时覆盖少量参数
-> 启用
-> 查看最近运行
```

## 低保真线框

## 1. 脚本模板工作台

### 页面结构

```text
[Header]
  标题: 模板创作工作台
  会话切换 | 新建会话 | 发布模板

[Step Bar]
  1 需求描述
  2 草稿编辑
  3 校验
  4 发布

[Main Layout]
  [Left 70%]
    当前步骤主内容

  [Right 30%]
    会话信息
    草稿版本
    参数协议摘要
    发布注意事项

[Bottom Status Strip]
  最近校验结果 | 最近发布结果
```

### 步骤展开

```text
Step 1 需求描述
[对话记录]
[本轮指令输入框]
[生成草稿]

Step 2 草稿编辑
[文件路径]
[语言]
[编辑器]
[保存当前草稿] [另存新版本]

Step 3 校验
[校验模式]
[基础校验]
[高级执行测试 (折叠)]
[校验结果摘要]

Step 4 发布
[模板名称 / 目标仓库 / 分支]
[提交信息]
[发布前摘要]
[确认发布]
```

### 设计意图

- 让用户明确知道当前处于创作哪一阶段
- 把“对话、编辑、校验、发布”从平铺改成阶段切换
- 把说明信息和历史信息放到右侧辅助区，不挤占主创作区

## 2. 任务中心

### 页面结构

```text
[Header]
  标题: 执行中心
  主按钮: 新建任务

[Top Strip]
  常用执行入口卡片
  - 交换机基线
  - 配置备份
  - F5 主线
  - 常用 smoke

[Main Layout]
  [Left]
    筛选条
    最近任务列表

  [Right]
    任务详情 / 日志
```

### 新建任务 Drawer

```text
[Drawer Title]
  新建任务

[Stepper]
  1 选择来源
  2 选择目标
  3 参数配置
  4 执行环境
  5 确认执行

Step 1 选择来源
  [模板选择]
  [或手工脚本入口]
  [模板摘要]

Step 2 选择目标
  [项目/分组]
  [区域]
  [Inventory 或 分组来源]

Step 3 参数配置
  [变量组]
  [Extra Vars]
  [必要参数]
  [高级参数: 折叠]

Step 4 执行环境
  [Controller / Agent]
  [凭证]
  [提权策略]

Step 5 确认执行
  [本次执行摘要]
  [风险提示]
  [确认执行]
```

### 详情区结构

```text
[任务状态卡]
  状态 | 模板 | 项目 | 开始时间 | 结束时间

[Tab]
  日志 | 上下游任务 | 执行摘要

[日志区]
  日志流
  来源过滤
  下载
  停止任务
```

### 设计意图

- “执行”是这个页面的绝对主任务
- 新建任务必须从长表单改成顺序决策
- 详情和日志常驻右侧或抽屉区，避免列表跳进跳出

## 3. 任务模板管理

### 页面结构

```text
[Header]
  标题: 模板目录
  主按钮: 新建模板

[Filter Bar]
  搜索 | 类型 | 场景标签

[Template List]
  每行/卡显示:
  - 名称
  - 用途说明
  - 类型
  - 默认入口文件
  - 默认执行方式
  - 变量组 / 仓库 / 凭证摘要
  - 操作: 跑一次 | 建计划 | 编辑
```

### 编辑结构

```text
[基础信息]
  名称 | 描述 | 类型

[默认执行配置]
  默认仓库 / 分支
  默认入口
  默认参数
  默认目标来源

[高级默认项]
  Agent / 凭证 / 高级参数
```

### 设计意图

- 把模板从“字段对象”变成“可读的能力目录”
- 优先支持“选模板”，其次才是“改模板”

## 4. 定时任务管理

### 页面结构

```text
[Header]
  标题: 调度中心
  主按钮: 新建计划

[List]
  名称 | 模板 | 项目/区域 | Cron | 启用状态 | 最近运行 | 操作
```

### 编辑结构

```text
[基础信息]
  名称
  模板
  项目/区域

[调度策略]
  Cron
  启用/禁用

[继承配置预览]
  从模板继承了什么

[高级覆盖配置 (折叠)]
  Inventory 覆盖
  Extra Vars 覆盖
  Agent / 凭证覆盖

[最近运行]
  最近任务 ID
  最近状态
  跳到任务中心
```

### 设计意图

- 让用户先看到“调度”而不是“执行字段”
- 把覆盖项降级成高级配置

## 页面风格原则

这组页面适合走 `product register + restrained color strategy`，也就是：

- 视觉上克制，不做营销页式大视觉
- 信息密度高，但必须有秩序
- 强化区块边界、步骤感、摘要感
- 不要堆太多卡片
- 不要靠大量提示文案解决结构问题

## 我建议的实现顺序

### Phase 1

先改 `任务中心`

原因：

- 这是日常使用频率最高的页面
- 最能体现“简洁、专业、易用”
- 也最能验证新的执行流程结构是否成立

### Phase 2

改 `脚本模板工作台`

原因：

- 它是当前第二个认知负担最高的页面
- 适合改成阶段式创作工作台

### Phase 3

改 `任务模板管理` 和 `定时任务管理`

原因：

- 这两个页面更像承接页
- 可以在前两者结构稳定后统一它们的字段组织方式

## 需要你确认的点

1. `任务中心` 是否认定为第一主入口
2. `脚本模板工作台` 是否接受改成 stepper / 分阶段工作台
3. `定时任务管理` 是否接受把大部分执行字段折叠到高级配置
4. 你更偏好列表式后台工具，还是“左列表右详情”的专业控制台模式
