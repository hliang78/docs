布局原则

  • 上半区负责 triage 和 analytics。
  • 左下区负责 module-oriented operations。
  • 右下区负责 compact health cluster。
  • `L2 Summary` 和 `Routing Summary` 不能漂到底部变大卡片。

实现提示

  • 外层用 24 列 CSS Grid。
  • 复合区内部再开子 Grid，不要靠绝对定位拼接。
  • 移动端可以纵向折叠，但桌面端必须保持当前扫描路径。
  • 这是通用布局展示，不承载 CE168 专有标题或槽位名。

文件用途

这个页面用于把规格文档里的抽象布局，直接转成可视化网页参考。 它适合在浏览器里对照 spec 讨论区块是否放对，而不是作为最终 Grafana 成品主题。
