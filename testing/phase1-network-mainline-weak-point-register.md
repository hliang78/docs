## 2. 分类口径

问题类型建议固定落在下面几类：

1. 采集缺口
2. 监控缺口
3. 拓扑漏边
4. 拓扑重边
5. 拓扑错边
6. 管理平面混入
7. 设备模型边界
8. 模板 / Agent 能力不匹配

使用规则：

1. 一条问题只描述一个最小可行动问题
2. `当前判断` 要区分“平台缺陷”“设备边界”“待补证”
3. `下一步动作` 必须能被后续任务直接消费，不写空泛表述

生命周期固定词表：

1. `开放`
2. `已缓解`
3. `已关闭`

## 4. 维护规则

1. 新问题继续追加 `W-008`、`W-009` 等编号
2. 问题再次出现时，必须同步更新 `最近出现运行`，并按实际情况调整 `当前状态`
3. 问题被缓解或关闭时，不删除原记录，必须同时更新 `当前状态`、`最近出现运行`、`当前判断` 和 `下一步动作`
4. 如果某问题已经证明是明确设备边界，也保留编号，避免后续重复开洞
